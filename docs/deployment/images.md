# Cordum container images

Every Cordum service ships as a multi-arch (linux/amd64 + linux/arm64)
container image published on each git tag push by
`.github/workflows/docker.yml`. Images are mirrored to GitHub Container
Registry (GHCR) and Docker Hub, signed with [cosign] keyless OIDC, and
shipped with SBOMs and provenance attestations generated by
`docker/build-push-action`.

[cosign]: https://docs.sigstore.dev/cosign/overview/

## Tag policy

| Tag | Source | Promotion rule |
|---|---|---|
| `:1.2.3` | `git tag v1.2.3` | Pinned, immutable. Reference this in production. |
| `:1.2` | `git tag v1.2.3` | Latest patch in 1.2.x. Updated on every patch release. |
| `:latest` | `git tag v1.2.3` | **Stable releases only.** Pre-release tags (`v1.2.3-rc.1`, `-beta`, `-alpha`) explicitly do **not** promote to `:latest` (gated in `docker/metadata-action`). |
| `:main` | push to `main` | Floating dev tag rebuilt on every commit to main that touches code. **Unsigned**, GHCR-only, not on Docker Hub. |
| `:sha-abcdef0` | push to `main` | Pinned floating dev tag for a specific commit. |

## Multi-arch pulls

Both `linux/amd64` and `linux/arm64` are available for every image. Docker
selects the right manifest automatically, so `docker pull` works the same
on:

* Apple Silicon (M-series Macs) — pulls `arm64`.
* AWS Graviton (EC2 c7g, m7g, r7g) — pulls `arm64`.
* Standard x86_64 hosts — pulls `amd64`.

To explicitly inspect or pull one arch:

```bash
docker manifest inspect ghcr.io/cordum-io/cordum/api-gateway:1.2.3
docker pull --platform linux/arm64 ghcr.io/cordum-io/cordum/api-gateway:1.2.3
```

## Cosign verification

All release-tag images are signed via cosign keyless mode using the
GitHub OIDC issuer. Verify before deploying to production:

```bash
TAG=1.2.3
IMAGE=ghcr.io/cordum-io/cordum/api-gateway:${TAG}

cosign verify "${IMAGE}" \
  --certificate-oidc-issuer https://token.actions.githubusercontent.com \
  --certificate-identity-regexp 'https://github\.com/cordum-io/cordum/\.github/workflows/docker\.yml@refs/tags/v.*'
```

Successful verification prints a JSON object with `optional.Bundle.Payload`
and a non-zero exit code on failure. The `--certificate-identity-regexp`
ensures only tag-triggered runs of our publish workflow are accepted —
forks and dev-branch builds are rejected.

`:main` and `:sha-*` tags are **not** signed because they are dev-grade
floating tags. Do not deploy them to production.

## Approximate image sizes

| Image | Compressed | Notes |
|---|---|---|
| `api-gateway`, `scheduler`, `safety-kernel`, `workflow-engine`, `context-engine`, `mcp`, `cordumctl` | ~25–35 MB | Static Go binary on `gcr.io/distroless/base-debian12:nonroot`. |
| `dashboard` | ~50 MB | nginx-unprivileged + the built React bundle. |

A complete pull of the full stack from a cold machine downloads ~250 MB.
`cordumctl` is also published as a container image for CI pipelines that
need the binary without a Go toolchain — pull `ghcr.io/cordum-io/cordum/cordumctl:latest`
or `docker run --rm -it cordum/cordumctl:latest --help`.

## Troubleshooting

* **`unauthorized` on GHCR pull.** Until the maintainers flip a package's
  visibility to public, anonymous pulls are denied. See
  [docs/deployment/ghcr-public-access.md](ghcr-public-access.md) for the
  one-time admin checklist.
* **Docker Hub rate-limit (429).** Anonymous pulls from Docker Hub are
  rate-limited to 100 requests / 6 h per IP. Either authenticate
  (`docker login`) or pull from GHCR instead — GHCR has no equivalent
  anonymous limit.
* **`no matching manifest for windows/amd64`.** All images are Linux only.
  Use Docker Desktop's WSL2 backend on Windows.
* **Image is `:dev` and pull fails.** A locally-built image whose
  `buildinfo.Version` was not injected falls back to `:latest`. Pin
  `CORDUM_VERSION=<tag>` in `.env` (or export it) before
  `docker compose pull` to force a specific release.
